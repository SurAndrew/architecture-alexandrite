# 2 «Мотивация»
 - Логи помогают при отладке разных компонентов приложения, а также позволяют собирать и 
 анализировать данные о работе системы. Это помогает обнаружить ошибки. 
 Ещё логи — это один из источников информации для мониторинга.
 - Логи помогают сформировать представление 
 о состоянии приложения. Если использовать эти подходы вместе, они смогут компенсировать 
 ограничения друг друга и получить более полную картину. 

Технические метрики
 - Количество запросов/ответов в еденицу времени через API
 - Количество этапов получения услуги (в трейсниге)
 - Среднее время отработки запросов API
 - Глубина просмотра сайта магазина
 - Время, проведенное клиентом в магазине


Бизнес-метрики
 - Количество клиентов магазина
 - Количество внешних клиентов API
 - Количество заказов
 - Количество повторных заказов
 - Средняя стоимость заказа


# 3. Предлагаемое решение
  Я бы поставил простую логособиралку, наиболее универсальную. Условно Всеядную. 
    - Graylog 
    - и/или Fluentd 
    - и/или LOGalyze. 
  А возможно и сразу несколько.  В зависимости от мощностей и возможнойстей команды.
  ОИБ логгирования не собо отличается от предложений для трейсинга...
  - Потребуется присоединение к существующей системе аутентификации комнании (если есть). (LDAP к примеру)
    - Разрабатывать свою не хотелось бы 
  - Разделение полномочий
    - Зайти в систему смогут только сотрудники компании с 
  актуальной учетной записью и ролью "Поддержка"
    - Разделение по VLAN между сайтом магазина и внутренней системой
    - Аутентификация для внешних Заказчиков
    - Аудит работы внешних и внутренних клиентов
    - Построение SSL аутентификации
    - Организация контролируемой зоны (Органихационно-технические меры)
    ... другие меры. Их много может быть
  
  Политика хранения логов
    - Шифрование - не обязательно до момета промышленной эксплуатации.
    - Управление доступом - ролевая модель
    - Логирование в реальном времени - на элементы API
    - Централизованное хранилище логов - да
    - Создание резервных копий - ежесуточное
    - Ротация файлов логирования - по времени (сутки), без очистки
